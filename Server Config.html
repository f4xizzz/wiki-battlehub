    Server Config

    General Configuration

    The server_config.json file is the core configuration for Cobblemon BattleHUB. It is generated automatically on the mod’s first startup inside your server directory.

    File Path

    config/cobblemon_battlehub/server_config.json

    Detailed Parameter Explanation

    1. Active Categories (Ladders)

    • activeRankedLadders: List of competitive ladder IDs (Ranked) available for server queues.
    • activeCasualLadders: List of non-ranked ladder IDs (Casual) available for quick matchmaking.

    2. Timers and Anti-AFK System

    • actionTimerSeconds (Default: 30): Maximum number of seconds a player has to make a decision during their battle turn.
    • afkStrikeLimit (Default: 3): Maximum number of turn timer failures before the player receives penalties.
    • afkExtensionSeconds (Default: 15): Additional seconds granted by the system in specific freeze or reconnection scenarios.
    • maxAfkAutopilotRounds (Default: 3): Maximum consecutive turns the autopilot system can act automatically before removing the player for AFK.
    • autopilotActionSeconds (Default: 5): Autopilot response time after the player’s timer fully expires.

    3. Seasons

    • currentSeasonNumber: Incremental counter identifying the current season number.
    • currentSeasonId: Internal ID generated automatically by the mod logic for storage (e.g., season_1).
    • currentSeasonName: Display name of the active season (e.g., Season 1).
    • currentSeasonStartedAtMs: Unix timestamp in milliseconds marking the exact start of the current season.
    • completedRankedSeasons: History storage for all finished seasons archived by /bh season rollover.

    4. Rewards System (rewards)

    Allows automating the delivery of items, currency, or permissions via console when goals are reached.

    Global Settings

    • milestoneRewardsEnabled: (true/false) Enables or disables player milestone rewards.
    • seasonRewardsEnabled: (true/false) Enables or disables rewards given to top placements at season end.

    Progress Milestones (milestoneRewards)

    Array where you register specific in-game goals:

    • id: Unique string to record the milestone delivery and prevent duplicate claims.
    • title: The display name shown to players.
    • statType: The statistic type evaluated for the trigger. Supported options in the Java class are:
    • TOTAL_BATTLES: Total battles fought (Ranked + Casual).
    • RANKED_WINS: Total wins in competitive queue.
    • RANKED_MATCHES: Number of matches played in competitive queue.
    • QUICK_WINS: Wins in casual matches.
    • QUICK_MATCHES: Casual matches completed.
    • threshold: The target number of wins or matches required to unlock the reward.
    • commands: List of console commands to run. The %player% placeholder is automatically replaced with the player’s name.

    Season End Rewards (seasonEndRewards)

    • placementRewards: List of rewards for top-ranked players.
    • maxPlacement: Maximum eligible placement (e.g., 1 applies only to Top 1, 3 applies to Top 2 and Top 3, etc.).
    • minimumGames: Minimum number of matches completed in the season to prevent players who only played one match from claiming Top 1.
    • commands: List of console commands executed for the winners.
    • participationReward: Consolation reward for participation.
    • minimumGames: Minimum number of matches the player must have completed to qualify for this participation reward.
    • commands: Commands sent to the console to reward eligible participants.
