    Ranks and Seasons System

    Cobblemon BattleHUB has a sophisticated integrated competitive system for ranking and pairing players. This system ensures matches are fair, discourages intentional abandonment, and automates season transitions.

    The Rating Algorithm (Glicko-2)

    Unlike the classic Elo system (which uses only a single fixed numerical score), BattleHUB uses the Glicko-2 algorithm to calculate players’ true skill with extreme precision.

    The system works with two main variables stored in the database for each category (Ladder):

    1. Rating (): The player’s estimated skill score (starting default: ).
    2. Rating Deviation (): The rating deviation, which measures how uncertain the system is about the player’s skill level (starting default: ).

    Desertion Penalty System (Leaver Buster)

    To prevent players from ruining the competitive experience by disconnecting from unfavorable battles, the CobblemonBattleHandler class monitors intentional abandons (Alt+F4, freezes, or crashes) and turn timeouts due to absence (AFK).

    What Happens in Case of Abandonment?

    If a player disconnects or is punished for inactivity during a competitive match, the server triggers the recordForfeitOrAFK(loser) method:

    1. Automatic Loss Assignment: The player who abandoned receives an immediate loss and loses Rating () as normal.
    2. W.O. Victory for the Opponent: The active player receives a message informing them the opponent abandoned the match and gains the victory points.
    3. Temporary Ban (Leaver Ban): The deserter is prohibited from entering the competitive queue for a cumulative period.

    When trying to re-enter the queue with an active ban, the player receives a dynamically formatted warning:

    Warning Example: "You are banned from the competitive queue for another 01h 15m due to recent abandons."

    Profile Data History

    All statistics are stored cleanly in the MySQL or SQLite database table. The monitored progress includes:

    • TOTAL_BATTLES: Number of matches played (Ranked + Casual).
    • RANKED_WINS: Total competitive wins.
    • RANKED_LOSSES: Total competitive losses.
    • RANKED_MATCHES: Total competitive matches completed.
    • QUICK_WINS and QUICK_MATCHES: Casual matchmaking stats.
    • RANKED_BAN_EXPIRATION: Timestamp in milliseconds indicating when the player may return to the ranked queue if penalized.

    Season Management

    The server administrator can automate seasons via console or in-game admin commands.

    Ending a Season (Season Rollover)

    When you decide a season has ended, run the command:

    /bh season rollover

    This command automatically performs the following actions in the database:

    1. Determine Winners: Scans the rankings and generates Top 1, Top 3, and Top 10 rewards based on the commands configured in your server_config.json.
    2. Filter Inactive Players: Only players who met the minimumGames requirement for the season are eligible for rewards and included in the final rankings, preventing inactive accounts from holding top leaderboard positions.
    3. Archiving and Logging: The current season is saved into the historical data list (completedRankedSeasons).
    4. Reset Ranks (Soft Reset): The Rating () of all active players is pulled back toward the mean () and Rating Deviation () is reset to for the new season.
